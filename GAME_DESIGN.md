# GAME DESIGN

スマホブラウザ/PWAで遊ぶ、落ち物パズルのゲームデザインメモです。

この文書は、実装済みの仕様、調整意図、今後の判断材料をまとめるためのものです。正確なコード上の定義は `app.js` を参照します。

## 基本方針

- 片手でも遊びやすいスマホ操作を優先する。
- 公式テトリスそのものを再現するより、このアプリ独自の遊びやすさと爽快感を重視する。
- ただし、ミノの抽選、ホールド、回転補正、ライン消去ディレイなど、遊び心地に直結する基礎仕様は現代的な落ち物パズルに寄せる。
- 上級者だけが得をするボーナスを増やしすぎない。特にBlastゲージは、4ライン消しやBack-to-Backに特別ボーナスを付けない。
- モードごとの違いは、タイトル画面の短い説明だけで直感できるくらいに保つ。

## 共通ルール

- 盤面は10列 x 20行。
- ミノ抽選は7-bag方式。7種類を1セットとしてシャッフルし、偏りすぎを抑える。
- ホールドあり。1つのミノにつきホールドは1回まで。
- 次のミノは3つ表示する。
- ライン消去時には短いディレイを入れ、消えるラインが光る。
- ミノをロックした瞬間にパーティクルを出す。
- Tミノは回転補正により、Tスピン的な隙間入りをしやすくしている。
- Tスピン、Back-to-Back、コンボ、Perfect Clearはスコア演出として扱う。
- ゲームオーバー/クリア時は盤面を残したまま結果オーバーレイを表示する。
- ミノ色は設定で切り替え可能。`Color A` はOkabe-Ito系、`Color B` はPaul Tol系、`Color C` は明るめ調整、`Color D` は従来色。
- 仮SEは音声ファイルを使わず、Web Audio APIで短い電子音を生成する。

## 操作

- 左右スワイプ: ミノを左右移動。
- 右側タップ: 時計回り回転。
- 左側タップ: 反時計回り回転。
- 下スワイプ: ハードドロップ。
- 下方向へドラッグし続ける: ソフトドロップ。
- 上スワイプ: ホールド。
- 左右移動は誤操作を避けるため、動き始めに少ししきい値を置く。

## モード一覧

### Endless

標準的なエンドレスモード。

- 通常の7種類のミノを使う。
- ゴールなし。
- 10ラインごとにレベルアップ。
- 既存の標準速度曲線を使う。
- 長く遊び、スコアやライン数を伸ばすモード。

### 200 Lines

通常ルールで200ライン消すことを目指すモード。

- 通常の7種類のミノを使う。
- 200ラインでクリア。
- 10ラインごとにレベルアップ。
- Endlessと同じ標準速度曲線を使う。
- 「普通のルールで区切りのある達成」を担当する。

### Clean Up

お邪魔ブロックを掃除するモード。

- 開始時、床側にお邪魔ブロックの列がある。
- お邪魔ブロックを含むラインを消すと、Clean Up系実績の対象になる。
- すべてのお邪魔ブロックが消えると、短い停止演出を挟んで床側から新しいお邪魔列が追加される。
- お邪魔列の追加時に上側が詰まっている場合はゲームオーバー。
- レベル/速度上昇は現在なし。
- 「盤面を掃除していく」わかりやすさと、軽いミッション感を担当する。

### Long Line

縦に長いミノで遊ぶ変化球モード。

- Iミノ相当の縦長ミノが5ブロックになる。
- ゴールなし。
- レベル/速度上昇は現在なし。
- 標準ルールとは違う大きな消去感を担当する。
- 名前は現在 `Long Line`。過去案の `5 Lines` より意味が伝わりやすい想定。

### Blast

ゲージをためて盤面を爆発的に立て直すモード。

- 通常の7種類のミノを使う。
- ゴールなし。
- 10ラインごとにレベルアップ。
- Blast専用のゆるめ速度曲線を使う。
- 左側、Hold下にBlastゲージを表示する。
- ゲージが80%以上になると予兆発光する。
- ゲージが100%になると、次の安定した区切りでBlast効果が発動する。
- ゲージ発動時にはゲージ周囲の発光/火花演出を出す。

#### Blastゲージの増加

- ライン消去: 1ラインあたり2%。
- 50ライン消せば、ライン消去だけで100%になる。
- ハードドロップ: 落下距離に応じて少し増える。
- ソフトドロップ: 下へ進むたびに少し増える。
- 盤面のブロックが半分より高い位置にある間、自動で毎秒0.5%増える。
- 4ライン消し、Back-to-Back、Tスピン、コンボによるゲージ追加ボーナスはない。

#### Blast効果

ゲージが100%になったとき、盤面状態に応じて効果が分岐する。

- ブロックで囲まれた空間がある場合: その空間に爆弾ブロックを1つ生成する。
- 囲まれた空間がない場合: 一番下の2行を丸ごと削除し、上から空行を足す。

爆弾ブロックを含むラインを消すと、そのラインの上下2行も巻き込んで消す。最大で5行消える。

爆弾が生成されたことでラインが完成した場合も、通常のライン消去判定に乗り、爆弾効果で周辺行を巻き込む。

## 速度曲線

### 標準速度曲線

Endlessと200 Linesで使う。

```text
Lv 1-20:
820, 760, 700, 640, 590, 540, 500, 460, 430, 400,
380, 360, 340, 320, 300, 285, 270, 255, 240, 225 ms
```

### Blast速度曲線

Blast専用。救済要素があるため、標準より少しゆるい。

```text
Lv 1-20:
700, 670, 640, 610, 580, 550, 525, 500, 475, 450,
430, 410, 390, 370, 350, 335, 320, 305, 290, 275 ms
```

## 実績の考え方

- 実績は「自然に遊んでいるうちに増える」ことを重視する。
- プレイ日数、累計ライン数、モード別プレイ回数、Clean Upライン数、Blastゲージ発動回数などを記録する。
- Blastゲージ実績は、ゲージが100%になって効果が発動した時点で1回として数える。
- ゲームオーバー/クリア時に、取得した実績をバナーで通知する。
- 実績画面では、取得年月日と進行状況を見られる。

## 未決・調整候補

- Clean Upの難易度。お邪魔列の追加量や穴の数が適切か。
- Long Lineに速度上昇を入れるか。
- Blastの速度曲線が簡単すぎる/忙しすぎるか。
- Blastゲージの80%予兆が早すぎる/遅すぎるか。
- スコアを今後どのくらいゲーム性の中心に置くか。
- ステージ選択画面を将来的に作るか。現状はタイトル画面にモードボタンを並べる。

## 更新時の注意

- モードを追加したら、`modes`、タイトル画面のボタン、モードアイコン、実績定義、Service Workerキャッシュを確認する。
- 画像アセットを追加したら、PWAで古い画像が残らないよう `sw.js` のキャッシュ名を更新する。
- スマホ操作を変えたら、Androidの戻るジェスチャー、iPhoneのPWA表示、片手操作の誤入力を確認する。
- 実績の保存形式を変えるときは、既存の `localStorage` データとの互換性を保つ。
